33.搜索旋转排序数组

假设按照升序排序的数组在预先未知的某个点上进行了旋转。

( 例如，数组 [0,1,2,4,5,6,7] 可能变为 [4,5,6,7,0,1,2] )。

搜索一个给定的目标值，如果数组中存在这个目标值，则返回它的索引，否则返回 -1 。

你可以假设数组中不存在重复的元素。

你的算法时间复杂度必须是 O(log n) 级别。

示例 1:

输入: nums = [4,5,6,7,0,1,2], target = 0
输出: 4
示例 2:

输入: nums = [4,5,6,7,0,1,2], target = 3
输出: -1

二分查找

思路：首先要求是O(logn)的时间复杂度，故为二分查找法。但二分查找法只针对于有序数组，故要把这个旋转后的数组拆分为两个单调的子数组。拆分的关键是找到该数组的最小（或最大）值，找最小（最大）值也可以使用二分查找法。

总结起来，整个程序有三次二分查找。

  • 第一次，找最小（最大）值。结束条件是子数组是单调递增的，最小（最大）值就是该子数组的头（尾）。该最值将原数组分为两个单调子数组。
  • 第二次，在第一个单调子数组中找是否有目标值。
  • 第三次，在第二个单调子数组中找是否有目标值。

34.在排序数组中查找元素的第一个和最后一个位置

给定一个按照升序排列的整数数组 nums，和一个目标值 target。找出给定目标值在数组中的开始位置和结束位置。

你的算法时间复杂度必须是 O(log n) 级别。

如果数组中不存在目标值，返回 [-1, -1]。

示例 1:

输入: nums = [5,7,7,8,8,10], target = 8
输出: [3,4]
示例 2:

输入: nums = [5,7,7,8,8,10], target = 6
输出: [-1,-1]

要求时间复杂度O(logn)，同样的二分查找法。。。。今天的这两道二分查找法的题要把我搞疯。。。。

做完这道题，我深刻体会到，二分查找只是一个思想，你想要找什么是可以通过你定义判断条件去自定义的。

比如这道题，要找的是某个数的起始位置和结束位置。可以用两次二分查找法，一次找起始位置，一次找结束位置。

  • 起始位置：如果nums[mid] != target，按常规二分查找法逼近。当nums[mid] == target时，令r = mid（从右边逼近，因为要找的是在靠左的位置），直到l != r结束。
  • 结束位置：注意，找结束为止时设置的：l=0, r=nums.size()。如果nums[mid] != target，按常规二分查找法逼近。nums[mid] == target时，从左边逼近（因为要找的是在靠右的位置），直到l != r结束。这时l - 1即为所求结果。
