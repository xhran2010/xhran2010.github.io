48.旋转图像

给定一个 n × n 的二维矩阵表示一个图像。

将图像顺时针旋转 90 度。

说明：

你必须在原地旋转图像，这意味着你需要直接修改输入的二维矩阵。请不要使用另一个矩阵来旋转图像。

示例 1:

给定 matrix =
[
[1,2,3],
[4,5,6],
[7,8,9]
],

原地旋转输入矩阵，使其变为:
[
[7,4,1],
[8,5,2],
[9,6,3]
]

49.字母异位词分组

给定一个字符串数组，将字母异位词组合在一起。字母异位词指字母相同，但排列不同的字符串。

示例:

输入: [“eat”, “tea”, “tan”, “ate”, “nat”, “bat”],
输出:
[
[“ate”,”eat”,”tea”],
[“nat”,”tan”],
[“bat”]
]

这道题的关键是找出这些互为异位词的词之间有什么联系。不难发现，互为异位词的词字母出现的种类和次数是一致的。所以我们可以想到，统计各异位词的单词的种类和出现的次数，如果一致，则放在一起；否则分开。所以可以用哈希表。

接下来就要解决怎么识别单词的种类和次数，由此衍生出以下两种算法。

用数字代替字母

一开始的想法是，从1开始给每个字母赋1个数字，如果和相等则互为异位词。代码也写好了，结果发现不通过。恍然大悟：$7+5=8+4$！

接下来继续想，加法不行，乘法可以吗？貌似还是不行：5*6=3*10。到这我就想不下去了，就想到第二种方法去了。但看了解答之后，发现这种方法还是可以继续想下去的。其实我忽略了一个很重要的隐含条件，就是字母的个数是有限的：只有26个。所以我可以不用程序去分配数字，而可以我预先定义好他们对应的数字了。乘法看似是不能唯一表示异位词的，但是我们是否可以回避这种多个因数的情况呢？答案是：素数！（数学功底不扎实的下场——想不到😅）两个素数相乘得到的数，由于素数不可以再分解质因数，故不可能产生其他组合的情况，相乘的结果就唯一了！

所以解题思路就产生了：用26个素数表示这26个字母，用对应的数的乘法表示单词，相乘结果相同的互为异位词。

设向量记录字母出现次数

因为字母的种类是有限的（26种），我们可以设长度为26的向量[0, 0, 0, …,0]表达a, b, c,… ,z的出现次数，向量相等的单词互为异位词。

故，我们可以创建map<int[26],int>，key表示该向量，value表示该向量对应的异位词在答案矩阵中对应的行数。代码不再给出。
