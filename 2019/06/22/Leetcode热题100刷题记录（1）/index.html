19.删除链表的倒数第n个节点

给定一个链表，删除链表的倒数第 n 个节点，并且返回链表的头结点。

示例：

给定一个链表: 1->2->3->4->5, 和 n = 2.

当删除了倒数第二个节点后，链表变为 1->2->3->5.

两遍扫描

​ 关键的问题是这道题并没有给出链表的长度，所以最简单的做法，也是我最想到的就是第一遍扫描确定长度，第二遍扫描找到倒数第n个节点。按链表的删除方法删除即可。

​ 这版代码质量并不高，单独列出了很多边界情况的判断。后续也许可以对该代码做优化。

双指针法

​ 更优的解法，是用两个指针可以实现一遍扫描。具体做法是：两个指向开头的指针，第一个先向后移动n个节点，然后两个指针一起向后移动。直到第一个指针移动到链尾，第二个指针指向的位置即为倒数第n个节点。其实这是一个很简单的思路，就好比一把长为n的尺子。

21/23.合并k个有序链表

​ 这其实是两道题，合并2个和合并k个。有相似之处，这里我放在一起解决。

合并2个有序链表

将两个有序链表合并为一个新的有序链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

示例：

输入：1->2->4, 1->3->4
输出：1->1->2->3->4->4

迭代

​ 思想也很简单，比较容易想到。即开一个新的指针，每次比较这两个有序链表的头结点的大小，把小的那一个串到新指针上去，直到两个有序链表为空。

递归

​ 我不太容易想到这个方法😬，个人感觉这个题拿递归做有点大材小用了😅，具体解法请参见官方题解。

合并k个有序链表

合并 k 个排序链表，返回合并后的排序链表。请分析和描述算法的复杂度。

示例:

输入:
[
1->4->5,
1->3->4,
2->6
] 输出: 1->1->2->3->4->4->5->6

迭代

​ 方法和合并2个有序链表很类似！区别仅在于每次要扫描一遍这k个链表的头结点的值。找到最小后，同样地将其串到新开的指针p后面去。

优先队列优化

​ 但是实际提交之后发现运行时间并不理想，时间主要耗费在每次遍历k个链表找最小的头结点，时间复杂度为$O(kN)$。而且这个时间浪费得也是不必要的，每次只会产生一个节点的更新，其他$k-1$个结点都还是原来的。其实这里我就应该想到用优先队列了，因为之前还专门看到过这方面的内容，奈何水平不够啊！

​ 优先队列和队列比较类似，唯一的区别在于每次取队首的元素都是这个队列中最小的元素（或最大，可以自己重载操作符）。我们首先建立一个优先队列把所有链表的头结点入队，每次出队1个结点，并将其后继结点（如果有的话）入队。以此类推，直到所有链表都为空。

下一个排列

实现获取下一个排列的函数，算法需要将给定数字序列重新排列成字典序中下一个更大的排列。

如果不存在下一个更大的排列，则将数字重新排列成最小的排列（即升序排列）。

必须原地修改，只允许使用额外常数空间。

以下是一些例子，输入位于左侧列，其相应输出位于右侧列。
1,2,3 → 1,3,2
3,2,1 → 1,2,3
1,1,5 → 1,5,1

​ 这道题中的“字典序”让我不是很理解，其实就是全排列的时候通常列举的顺序，但是我总结不出一般的规律。题目中说到如果不存在下一个更大的序列，那么这个序列指的就是降序序列。在网上查了一下，这道题的算法是：

  • 从末尾开始找$i$，直到$num[i]<num[i+1]$，确定$i$的位置。
  • 再从末尾开始找$j$，直到找到第一个$num[j]>num[i]$，确定$j$的位置。
  • 交换$i$和$j$，新的位置记为$i’$和$j’$
  • 把$num[j’]$~$num[-1]$反转一次

​ 为什么可以想到这个算法呢？我觉得可以从以下几点考虑

  • 字典序的最后一个是完全的降序，也就是倒过来是完全的升序

  • 那么如果不是字典序的最后一项，打破他的情况只可能是倒序时出现降序，而第一次出现降序时就为关键的位置

  • 要使字典序变得更大，需要找到一个比当前确定的位置更大的数，这个数来自于这个位置之后的一个数

  • 注意最后还要反转一次那个位置之后的序列，这个原理类似于从199到200的变化。百位+1了，十位和个位置零
